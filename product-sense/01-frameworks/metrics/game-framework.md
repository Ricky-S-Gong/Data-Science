# GAME Framework for Metrics Questions

## Use Case

Use GAME when asked to define success metrics, choose a north star metric, identify KPIs, or measure the success of a product, feature, or company.

## Acronym

GAME:

- Clarify
- Goals
- Actions
- Metrics
- Evaluate

## 1. Clarify

Address ambiguity, state assumptions, and establish scope.

Ask:

- Are we measuring a feature, product, or company?
- Should we consider the parent company's goals?
- Are any related features in or out of scope?
- Are we choosing one north star metric or a set of metrics?

## 2. Goals

Define the main goal for the company, product, or feature.

Ask:

- What is the product vision?
- What does this product do for users?
- How does it support the company strategy?
- What does success mean in this context?

## 3. Actions

List user actions that support the goal.

Use a user journey map:

- Pick one use case.
- Walk through the user steps.
- Identify behaviors that prove the goal is being achieved.

## 4. Metrics

Translate key actions into measurable metrics.

Common categories:

- Product metrics
- OKRs
- Business metrics
- Quality metrics
- Leading indicators
- Counter or guardrail metrics

## 5. Evaluate

Assess whether the selected metric is good enough.

Ask:

- Where does this metric fall short?
- Could it be misleading?
- Does the metric match the feature scope?
- What counter metrics or guardrails should we track?

## Example: Instagram Discover Feed

Goal:

Help users develop and expand their interests.

Key actions:

- Open Discover regularly
- Open posts shown
- Interact with content
- Follow creators found
- Scroll for more content
- Search related topics

Potential metrics:

- Total posts opened by user
- Time spent looking at content by user per session
- Scroll depth per session per user
- Percentage of Discover sessions with a post click

Chosen metric:

Time spent looking at content by user per session.

Guardrail:

Pair time spent with engagement metrics such as likes or creators followed from Discover feed.
