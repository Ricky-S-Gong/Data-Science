# 如何回答指标类问题

## 关键术语

| 术语 | 含义 |
| --- | --- |
| PM | 产品经理。 |
| KPI | 关键绩效指标，用来衡量重要目标的推进情况。 |
| 北极星指标 | 最能概括产品或业务成功的单一核心指标。 |
| GAME 框架 | 澄清问题、定义目标、梳理用户行为、选择指标、评估指标。 |
| DAU | 日活跃用户数。 |
| CTR | 点击率，通常等于点击次数除以曝光次数。 |
| OKR | 目标与关键结果。 |
| MTTR | 平均解决时间。 |
| 护栏指标 | 用来防止优化某个目标时伤害其他重要体验或业务结果的指标。 |
| 用户旅程图 | 描述用户为完成某个目标经过的步骤、页面、决策点和行为。 |

## 核心复习

不要一上来就说日活、留存或收入。先定义这个产品场景里的成功是什么，再把成功转化为用户行为和可衡量指标。

好的指标题答案要讲清楚：

- 产品或功能要达成什么目标。
- 哪些用户行为能证明目标正在达成。
- 哪些指标最能捕捉这些行为。
- 所选指标可能在哪里误导判断。
- 需要搭配哪些护栏指标。

## 指标题通常长什么样

在典型的分析类 PM 面试中，你可能会遇到两道指标题。每道题通常持续 20 到 25 分钟，其中包括追问。

指标题形式比较直接，但范围可能不同：

- 一个小功能。
- 一个产品。
- 整家公司。

面试官可能要求你定义：

- 一个北极星指标。
- 一组 KPI。
- 一套衡量功能、产品或公司成功的方法。

北极星指标是最能概括成功的单一指标。清晰的北极星指标有助于对齐相关方、保持以客户为中心，并让分析更简单。

常见问题包括：

- 为 Airbnb 定义一个北极星指标。
- 作为即将在 WhatsApp 上推出新功能的 PM，你会衡量哪些指标？
- 你如何定义成功指标？

常见追问包括：

- 除了北极星指标，还有哪些额外指标值得追踪？
- 你选择的指标有什么盲点或挑战？
- 你能想到哪些策略来推动这些指标？

## 面试官想看什么

面试官希望看到你能：

- 为功能、产品或公司设定目标。
- 根据当前场景发散出相关指标。
- 权衡不同指标的优缺点。
- 选择最适合目标的指标或指标组。
- 展示产品判断和战略推理，而不是只会背指标名。

扎实答案的关键，是先定义这个场景里的“成功”到底是什么。只有先理解你要衡量什么、为什么衡量，后面选指标才有依据。

## GAME 框架

GAME 是推进指标题的结构化方法。它能帮助你一边展开答案，一边为最终指标选择建立依据。

| 步骤 | 要回答的问题 | 产出 |
| --- | --- | --- |
| 澄清问题 | 题目范围到底是什么？ | 假设和边界。 |
| 定义目标 | 什么才算成功？ | 清晰的公司、产品或功能目标。 |
| 梳理用户行为 | 哪些用户行为支持这个目标？ | 有意义的用户行为列表。 |
| 选择指标 | 这些行为如何被衡量？ | 候选指标和最终指标。 |
| 评估指标 | 指标在哪里可能误导？ | 弱点、取舍和护栏指标。 |

## 贯穿示例：Instagram Discover Feed

假设面试官问：

“你会如何衡量 Instagram Discover feed 的成功？”

下面用 GAME 框架完整走一遍。

## 1. 澄清问题

展开答案前，先澄清题目中的模糊点，并说明自己的假设。这能确保你没有一开始就跑偏。

如果不知道该问什么，可以围绕这些方面澄清：

- 公司或母公司是否在范围内。
- 具体产品界面是什么。
- 相关功能是否包含在题目里。
- 面试官想要一个指标，还是一组指标。

对于 Instagram Discover 这道题，可以问：

- 我是否应该考虑 Instagram 母公司 Meta 的目标？
- Discover feed 里的搜索栏是否包含在这次分析里？

假设面试官回答：

- 需要考虑 Meta 的目标。
- 搜索栏暂时不在范围内。

![指标题白板：澄清问题](../../assets/images/analytical-questions/metrics/whiteboard-1-clarify.png)

## 2. 定义目标

接下来定义公司、产品或功能的主要目标。后面的答案都要不断回到这个目标。

可以思考：

- 产品愿景是什么？
- 这个产品为公司整体目标提供什么价值？
- 这个产品为用户提供什么价值？
- 它支持了公司的哪部分使命或战略？

如果你不知道公司官方使命，可以做一个合理假设，但要告诉面试官这是你的假设。

对于 Instagram Discover：

- Meta 的更大使命可能包括沟通、分享和社区建设。
- Discover 是一个基于兴趣的内容流，向用户展示他们原本不一定能发现的内容。
- 因此，Discover 通过让用户接触世界中新的部分，帮助用户拓展兴趣。

示例回答：

“我假设 Meta 的使命是赋予人们建立社区的能力。Discover feed 是一个基于兴趣的内容流，它向用户展示他们原本不会发现的内容。因此，我会把 Discover feed 的目标定义为：帮助用户发展并拓展兴趣。”

![指标题白板：定义目标](../../assets/images/analytical-questions/metrics/whiteboard-2-goals.png)

## 3. 梳理用户行为

目标清楚之后，找出哪些用户行为能证明目标正在达成。

可以问自己：

- 我们希望用户采取哪些行动？
- 哪些行为可以证明用户正在发展并拓展兴趣？

常见方法是走一遍用户旅程图。

### 用户旅程图

用户旅程图，也可以理解为用户漏斗，是用户为了完成某个目标所经历的完整流程。完整 walkthrough 可以包括每个页面、每个决策点和每次点击。

建立用户旅程图时：

1. 选择一个用户目标和产品目标一致的使用场景。
2. 从用户进入产品开始，走到这个旅程的终点。

不要为了省时间一次映射多个场景。那样很容易把不同行为混在一起。必要时，可以一个旅程一个旅程地讲。

### Instagram Discover 的用户旅程

使用场景：用户想在短时间内被新内容娱乐。

旅程：

- 用户打开 Instagram，看到首页信息流。
- 用户点击 Discover 标签页。
- 用户看到帖子和 Reels 组成的网格。
- 搜索功能存在，但这道题里暂时不在范围内。
- 用户继续滚动。
- 用户点击一个帖子或 Reel。
- 用户可能点赞、评论、访问发布者主页，或进行其他互动。
- 用户可能回到 Discover feed。
- 当用户退出 app 或切换到产品其他区域时，旅程结束。

能说明用户正在拓展兴趣的关键行为包括：

- 定期打开 Discover feed。
- 打开 Discover 展示的帖子。
- 对展示内容点赞、评论或分享。
- 关注在 Discover 中发现的创作者。
- 继续滚动查看更多内容。
- 搜索相关话题。

![指标题白板：用户行为](../../assets/images/analytical-questions/metrics/whiteboard-3-actions.png)

## 4. 把行为转化成指标

识别关键行为之后，开始发散能衡量这些行为的指标。

实用做法：

1. 先列出 5 到 10 个候选指标。
2. 解释每个候选指标捕捉了什么。
3. 排除过宽、噪音太大或和目标关联较弱的指标。
4. 根据题目要求，最后收敛到 1 到 3 个指标。

你还应该明确说明：哪些指标你考虑过，但最终没有选。

### 常见指标类型

| 指标类型 | 衡量什么 | 例子 | 注意点 |
| --- | --- | --- | --- |
| 产品指标 | 用户如何与产品互动。 | DAU、CTR、使用时长。 | 需要结合产品改动、季节性和外部事件解释。 |
| OKR | 战略目标和可量化关键结果。 | 特定市场新增用户、DAU 增长、流失下降。 | 适合团队对齐，但不一定反映整体业务健康。 |
| 业务指标 | 公司层面的高层表现。 | 收入、净利润率、客户生命周期价值。 | 往往是滞后指标，小功能很难直接影响。 |
| 质量指标 | 产品或流程是否正常运行。 | 缺陷积压量、MTTR、页面加载时间。 | 常用于诊断，不一定适合作为最终成功指标。 |
| 领先指标 | 变化更快的早期信号。 | 注册量、按钮点击。 | 灵敏但噪音大，需要和更稳定的指标一起看。 |
| 反向或护栏指标 | 某个目标是否伤害其他优先级。 | 流失率、客户满意度。 | 应该和主要指标一起追踪。 |

### Instagram Discover 候选指标

从用户旅程中选出的关键行为是：用户打开 Discover 中展示的帖子。

候选指标包括：

- 每位用户打开的帖子总数。
- 每位用户每次会话中查看内容的时长。
- 每位用户每次会话的滚动深度。
- 有帖子点击的 Discover 会话占比。

每个指标表达的信息略有不同。如果目标是判断用户是否正在发展兴趣，“每位用户每次会话中查看内容的时长”是一个有力候选，因为它也能捕捉兴趣深度。

![指标题白板：选择指标](../../assets/images/analytical-questions/metrics/whiteboard-4-metrics.png)

## 5. 评估指标弱点

所有指标都有强项和弱点。评估环节体现的是成熟的产品判断。

可以问：

- 这个指标在哪里会失效？
- 它是否会产生假阳性？
- 它是否会漏掉重要的用户体验问题？
- 它应该搭配哪些护栏指标？

对于 Instagram Discover：

- 选择的指标：每位用户每次会话中查看内容的时长。
- 弱点：用户花很多时间，可能是因为无聊、一直找不到感兴趣的内容，而不是因为内容有价值。
- 护栏：把使用时长和点赞、从 Discover 关注创作者等参与信号结合起来。

## 答案质量标准

| 水平 | 表现 |
| --- | --- |
| 普通 | 能大致解释产品目标，列出可行指标，并为北极星指标给出合理理由。 |
| 良好 | 能把产品目标连接到用户行为，再把用户行为映射到指标；也能说明指标可能误导的地方并提出护栏。 |
| 优秀 | 深刻理解产品为什么存在、用户是谁、战略角色是什么、市场语境是什么；北极星指标和产品价值及组织成功紧密相关。 |

## 常见错误

- 回答面试官真正问的问题。如果对方要求一个北极星指标，不要最后给三个并列选项。
- 确保答案是真正的指标。“参与度”不是一个具体指标，而是一类指标。
- 明确定义指标计算口径。如果指标是 DAU，要说明什么算活跃；如果指标是使用时长，要说明是按会话、按用户，还是按月计算。
- 让指标粒度匹配产品范围。小功能对整体 DAU 的影响很可能被噪音淹没。
- 不要只说北极星指标。还要讨论盲点、护栏和取舍。

## 面试答题模板

1. 澄清范围和假设。
2. 定义公司、产品或功能目标。
3. 梳理支持目标的用户行为。
4. 发散候选指标。
5. 选择最有代表性的指标或指标组。
6. 解释为什么它适合目标。
7. 评估弱点，并补充护栏指标。
