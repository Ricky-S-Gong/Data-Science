# 如何回答 A/B 测试类问题

## 关键术语

| 术语 | 含义 |
| --- | --- |
| A/B 测试 | 对照实验，用来比较当前体验和改动体验的效果。 |
| 产品经理（PM，Product Manager） | 负责产品目标、用户价值、优先级和落地执行。 |
| 点击率（CTR，Click-Through Rate） | 通常等于点击次数除以曝光次数。 |
| 对照组 | 看到当前体验或默认方案的用户组。 |
| 实验组 | 看到新设计、新功能或改动方案的用户组。 |
| 用户群组 | 按某种规则划分出来的一组用户。 |
| 曝光次数 | 某个元素、页面或广告被展示的次数。 |
| 跳出率 | 用户进入某页面后没有进一步互动就离开的比例。 |

## 核心复习

A/B 测试类问题考察的是你能否用实验验证产品决策。好的回答需要：

- 定义清晰假设。
- 描述精确实验设置。
- 选择主要指标、辅助指标和护栏指标。
- 说明如何根据结果做上线决策。
- 讨论指标不一定能完整捕捉的取舍。

## 五段式回答结构

| 步骤 | 核心问题 | 面试官想看什么 |
| --- | --- | --- |
| 假设 | 你在验证什么？ | 明确的因果判断，而不是模糊的改版想法。 |
| 方法 | 你会如何测试？ | 对照组、实验组、用户范围、随机分组和边界。 |
| 指标 | 你会追踪什么？ | 成功指标、诊断指标和护栏指标。 |
| 影响 | 你会如何使用这些信息？ | 上线、迭代或回滚的决策规则。 |
| 权衡 | 可能出什么问题？ | 数字之外的产品判断。 |

## 为什么会考这类题

A/B 测试是产品经理理解用户行为的核心工具之一。在很多大型科技公司里，产品经理会深度参与实验，因为实验能帮助验证产品决策。

因此，产品管理面试里经常出现这类问题：

- 你会在 Google 首页上做哪些实验来提升搜索查询量？
- 为了提升收入，你会优先在 Facebook 广告上做哪三类 A/B 实验？

实验类问题可以套结构，但优秀答案必须覆盖下面几个基本要素。

## 1. 假设

对于任何 A/B 实验，先告诉面试官你的假设是什么。你到底在验证什么？

例子：

- 较弱：“我们测试一个更大的按钮。”
- 更好：“我们认为增大按钮面积会提高点击率，因为用户更容易注意到主要操作。”

好的假设通常可以写成：

如果我们针对某类用户改变某个产品元素，那么某个关键指标应该会提升，因为这个改动解决了某个明确问题。

## 2. 方法

说完假设后，要说明实验里具体改什么。

按钮例子可以这样设置：

- 对照组：用户看到当前体验。
- 实验组：用户看到面积增加 50% 的按钮。

这里必须精确。面试官需要清楚理解你提出的方案是什么，以及实验如何设置。

方法部分应该覆盖：

- 用户范围：哪些用户进入实验。
- 用户分层：测试面向所有用户，还是某个特定群体。
- 随机分组：用户如何被分配到不同组。
- 实验时长：实验需要跑多久。
- 样本量：是否有足够用户产生可信结果。
- 排除项：是否排除内部用户、机器人或异常流量。

不要忘记对照组。没有对照组，数据无法说明观察到的变化是否真由改动造成。

## 3. 指标

实验设置清楚后，说明哪些指标重要。指标应该能给产品和工程团队提供有用洞察。

在按钮例子里，最明显的指标是点击率（CTR）。其他相关指标包括：

- 曝光次数。
- 页面上其他按钮的点击率（CTR）。
- 按钮悬停时间。
- 页面停留时间。
- 点击按钮跳转到新页面后的跳出率。

完整指标组通常包含：

| 指标层级 | 作用 | 例子 |
| --- | --- | --- |
| 主要成功指标 | 判断实验是否达成主要目标。 | 目标按钮点击率（CTR）。 |
| 辅助指标 | 帮助理解用户行为。 | 悬停时间或页面停留时间。 |
| 护栏指标 | 确认改动没有伤害其他重要结果。 | 其他按钮点击率（CTR）或跳出率。 |
| 诊断指标 | 帮助解释结果为什么变化。 | 曝光次数或分群表现。 |

## 4. 影响

实验本身只是提供信息。强答案要说明你如何使用这些信息。

可以提前给出决策规则：

- 主要指标提升且护栏稳定：上线或逐步扩大流量。
- 主要指标提升但护栏恶化：继续调查、迭代方案，或限制上线范围。
- 主要指标没有提升：不上线，修正假设。
- 结果混杂：做分群分析、延长实验、补充定性研究，或设计后续实验。

这也是把实验和公司整体愿景、产品目标连接起来的机会。

## 5. 权衡

每个重新设计或新功能都会有取舍。纯数据分析可能漏掉一些重要风险。

例如，把某个按钮变大可能提高这个按钮的点击率（CTR），但同时减少用户对页面上其他重要元素的注意力。

可以从这些角度讨论：

- 局部优化与整体体验。
- 短期指标变化与长期信任。
- 更多点击与更有意义的互动。
- 收入与用户满意度。
- 简洁性与可发现性。
- 指标盲点。

## 面试答题模板

1. 说明假设。
2. 定义对照组和实验组。
3. 说明实验用户范围和分层方式。
4. 选择主要指标、辅助指标、诊断指标和护栏指标。
5. 解释如何解读结果并做上线决策。
6. 讨论指标之外的取舍和风险。
