# Improving Facebook's DAU

## Key Terms

| Term | Meaning |
| --- | --- |
| DAU | Daily Active Users: the number of unique users who are active on a product in a given day. |
| Core action | A user behavior that reflects meaningful product value, such as messaging, reacting, posting, or viewing relevant content. |
| Engagement loop | A repeated cycle that brings users back, such as seeing friend activity, responding, and creating more activity for others. |
| Retention | The percentage of users who return after a given period, such as D1, D7, or D30. |
| Churn | The state where previously active users stop returning or become inactive. |
| Guardrail metric | A metric that protects user experience while optimizing DAU, such as notification opt-out rate or bounce rate. |

## Question

How can you improve Facebook's DAU?

## Hint

Think about what actions bring users back daily and how you can reinforce or expand those loops without relying only on notifications.

## Why This Question Matters

This is a classic product sense and metrics execution question.

The interviewer is not asking for a random list of growth hacks. They want to see whether you can:

- Define DAU precisely.
- Identify the user behaviors that naturally drive daily usage.
- Segment the DAU problem instead of treating all users the same.
- Propose experiments that improve durable product value.
- Avoid short-term tricks that raise DAU while hurting long-term trust or satisfaction.

For Facebook, DAU is a health metric because daily usage reflects habit, content relevance, social connection, and perceived value.

## Recall: The GAME Framework

The GAME framework helps structure thinking around user growth and engagement.

| Step | What It Means For This Question |
| --- | --- |
| Goals | Define what DAU means and what is driving it. |
| Actions | Identify key user behaviors that increase daily usage. |
| Metrics | Measure those behaviors and segment usage trends. |
| Evaluate | Design and test interventions for sustainable impact. |

## Step 1: Goals

The goal is to increase the number of unique users who return to Facebook each day.

Daily Active Users is a core health metric because it reflects:

- Habitual use: users have a reason to come back often.
- Content relevance: users see posts, videos, groups, or recommendations that feel worth their time.
- Social value: users maintain relationships through comments, messages, reactions, tags, and friend updates.
- User satisfaction: users feel Facebook is worth opening instead of ignoring.

The important nuance is that we should not drive DAU through gimmicks.

Bad DAU growth can come from:

- Spammy notifications.
- Dark patterns.
- Clickbait content.
- Low-quality viral loops.
- Features that increase opens but reduce trust.

Good DAU growth should come from increasing long-term retention and product value.

## Step 2: Actions

To improve DAU, first identify what users do when Facebook is valuable enough to open daily.

### Passive Consumption

Passive consumption means users consume content without necessarily creating anything.

Examples include:

- Scrolling the feed.
- Watching Stories.
- Watching Reels or videos.
- Reading comments.
- Browsing groups.
- Checking Marketplace or events.

This matters because many users return daily primarily to see what is new.

### Social Triggers

Social triggers are interactions caused by other people.

Examples include:

- Responding to comments.
- Replying to messages.
- Checking tags.
- Accepting friend requests.
- Seeing birthday reminders.
- Responding to group replies.

These are powerful because Facebook is a social network, not just a content feed. A notification about a real person is often more meaningful than a generic content recommendation.

### Creation And Contribution

Creation and contribution behaviors make the ecosystem healthier because they produce content and signals for others.

Examples include:

- Posting Stories.
- Sharing content.
- Reacting to updates.
- Commenting on friend posts.
- Creating group posts.
- Posting photos or life updates.

These actions matter because DAU is easier to sustain when users are not only consuming but also creating reasons for others to return.

### Notification-Driven Re-engagement

Notifications can bring users back, but they should not be the only mechanism.

Examples include:

- Push notifications.
- Email reminders.
- Badges.
- In-app reminders.
- Friend activity nudges.

The risk is notification fatigue. If Facebook relies too heavily on notifications, users may turn them off, bounce quickly, or develop negative sentiment.

### Users At Risk Of Churn

Improving DAU often means identifying users who used to be active but are drifting away.

Useful churn-risk signals include:

- Fewer sessions per week.
- Lower feed scroll depth.
- Fewer meaningful interactions.
- Longer time since last post, comment, or message.
- Lower notification response rate.
- Reduced friend or group activity.

For these users, the product should restore a genuine value loop instead of just sending more prompts.

## Step 3: Metrics

The main metric is DAU, but it should be segmented and supported by behavioral metrics.

### Primary Metric

Track:

- DAU: unique active users per day.

Define active clearly. For example, an active user could be someone who opens Facebook and performs a meaningful action, not merely someone who receives a background sync or accidental page load.

### Segment DAU

DAU should be segmented by:

- User cohort.
- New versus existing users.
- Resurrected users.
- Platform: iOS, Android, mobile web, desktop web.
- Region.
- Age of account.
- User interest area, such as groups, video, friends, Marketplace, or events.
- Notification permission status.

Segmentation matters because DAU problems are rarely uniform. A decline among new users may suggest onboarding issues, while a decline among long-time users may suggest feed relevance or social graph fatigue.

### Core Action Metrics

Track the frequency of core actions per user, such as:

- Messages sent.
- Stories viewed.
- Posts viewed.
- Reels watched.
- Comments made.
- Reactions sent.
- Friend requests accepted.
- Group posts viewed or created.
- Marketplace visits.
- Event responses.

These metrics help explain what is driving DAU, not just whether DAU moved.

### Notification Metrics

Track:

- Push notification click-through rate.
- Time to return after notification.
- Notification opt-out rate.
- Notification dismiss rate.
- Sessions generated per notification.
- Bounce rate after notification-driven return.

The point is not to maximize notification volume. The point is to understand which notifications bring users back to meaningful experiences.

### Retention Metrics

Track:

- D1 retention: whether new users return one day later.
- D7 retention: whether users return one week later.
- D30 retention: whether users return one month later.
- Reactivation rate among previously inactive users.

New user onboarding deserves special attention. If new users do not quickly find friends, groups, or content they care about, future DAU will be limited.

### High-Engagement Action Share

Track:

- Percentage of DAUs who perform at least one high-engagement action.

High-engagement actions could include:

- Sending a message.
- Posting a Story.
- Commenting.
- Joining or posting in a group.
- Sharing content.
- Responding to an event.

This metric helps distinguish shallow DAU from meaningful DAU.

### Churn And Inactivity

Track:

- Churn rate among previously active users.
- Inactivity rate by cohort.
- Percentage of users with declining weekly sessions.
- Percentage of users who have not performed a core social action in 7 or 30 days.

This helps identify where DAU is leaking.

## Step 4: Evaluate

To improve DAU, focus on low-effort, high-impact experiments that create durable value.

### Experiment 1: Better Onboarding

Hypothesis:

New users will return more often if they quickly understand the product's key value loops.

Possible changes:

- Help users find friends faster.
- Recommend relevant groups during onboarding.
- Ask users about interests and tune feed recommendations.
- Show users how Stories, groups, and messaging work.
- Encourage first meaningful action within the first session.

Metrics:

- D1, D7, and D30 retention.
- Time to first friend connection.
- Time to first meaningful interaction.
- New user DAU contribution.
- Onboarding completion rate.

Guardrails:

- Onboarding drop-off rate.
- User-reported confusion.
- Skip rate for recommended steps.

### Experiment 2: Smarter Personalized Notifications

Hypothesis:

Users are more likely to return when notifications are personalized to their interests, relationships, and time zones.

Possible changes:

- Send fewer but more relevant notifications.
- Prioritize notifications from close friends or active groups.
- Time notifications based on historical active hours.
- Avoid stacking too many notifications in a short period.

Metrics:

- Notification CTR.
- Time to return after notification.
- Session depth after notification.
- High-engagement action rate after return.

Guardrails:

- Notification opt-out rate.
- Notification dismiss rate.
- Bounce rate.
- User complaints or negative feedback.

### Experiment 3: Surface Relevant Friend Activity

Hypothesis:

Users will return more often when Facebook highlights meaningful activity from people they care about.

Examples:

- "3 friends posted a story."
- "A close friend commented on your post."
- "People in your group are discussing a topic you follow."
- "Friends are attending an event near you."

Metrics:

- Return rate after friend-activity nudge.
- Story views.
- Comments or reactions.
- Sessions per user.
- Percentage of DAUs with a social interaction.

Guardrails:

- Hide or mute rate.
- Perceived spamminess.
- Feed satisfaction.

### Experiment 4: Highlight Unfinished Engagement Loops

Hypothesis:

Users are more likely to return when Facebook makes unfinished social loops visible and easy to complete.

Examples:

- Unread messages.
- Pending friend requests.
- Replies to comments.
- Tagged photos awaiting review.
- Group posts with active replies.
- Drafted but unfinished posts.

Metrics:

- Completion rate of unfinished loops.
- Return frequency.
- Messages sent.
- Comments made.
- Friend requests accepted.

Guardrails:

- User frustration.
- Notification fatigue.
- Privacy concerns.

### Experiment 5: Improve Content Relevance

Hypothesis:

Users will return daily if the feed reliably shows relevant content instead of low-quality or repetitive content.

Possible changes:

- Better ranking for close-friend content.
- More relevant group posts.
- Less clickbait.
- More local or interest-based recommendations.
- More control over feed preferences.

Metrics:

- Feed session depth.
- Dwell time on quality content.
- Reactions and comments.
- Hides and unfollows.
- D7 and D30 retention.

Guardrails:

- Misinformation exposure.
- Low-quality content prevalence.
- User-reported feed satisfaction.

## How To Prioritize Experiments

I would prioritize interventions by:

- Impact on DAU.
- Confidence in the hypothesis.
- Ease of implementation.
- Risk to user trust.
- Whether the effect is likely to be durable.

| Experiment | Likely Impact | Risk | Why |
| --- | --- | --- | --- |
| Better onboarding | High for new-user DAU | Medium | Helps future DAU base, but requires careful flow design. |
| Smarter notifications | Medium to high | High | Can drive return, but overuse creates fatigue. |
| Friend activity surfacing | High | Medium | Leverages Facebook's social graph and reinforces core value. |
| Unfinished loops | Medium | Medium | Useful if loops are meaningful, annoying if overdone. |
| Content relevance | High | High | Deeply tied to long-term DAU, but harder to execute. |

## A/B Testing Plan

For each intervention:

1. Define the target segment, such as new users, at-risk users, or users active in groups.
2. Randomly assign users to control and treatment.
3. Predefine the primary metric and guardrails.
4. Run the test long enough to capture retention effects, not just same-day opens.
5. Analyze heterogeneous treatment effects by cohort, platform, and region.
6. Launch only if DAU improves without harming user satisfaction or trust.

## What A Strong Answer Emphasizes

A strong answer should not say: "Send more notifications."

A stronger answer says:

- DAU is a result of daily value loops.
- Facebook should improve the loops that naturally bring people back.
- Notifications can help, but only when they point to meaningful content or social interaction.
- DAU should be segmented so we understand whether the problem is onboarding, churn, content relevance, or social graph activity.
- Experiments should include guardrails for notification fatigue, bounce rate, and quality.

## 30-Second Answer

I would improve Facebook DAU by using the GAME framework. The goal is to increase unique users returning daily through real product value, not gimmicks. I would identify the key actions that drive daily usage: feed and video consumption, social triggers like comments and messages, creation such as Stories and posts, and re-engagement loops. I would track DAU by cohort, platform, and region, plus D1/D7/D30 retention, core actions per user, notification CTR, high-engagement action share, and churn among previously active users. Then I would test interventions like better onboarding, smarter personalized notifications, surfacing relevant friend activity, and highlighting unfinished engagement loops, while guarding against notification fatigue, bounce rate increases, and lower feed satisfaction.
