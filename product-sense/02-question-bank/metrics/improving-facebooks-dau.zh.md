# 提升 Facebook 的日活跃用户数

## 关键概念

| 概念 | 解释 |
| --- | --- |
| 日活跃用户数（DAU，Daily Active Users） | 指某一天内活跃使用产品的独立用户数量。 |
| 核心行为 | 能体现产品真实价值的用户行为，例如发消息、点赞、评论、发布内容或观看相关内容。 |
| 参与循环 | 能反复把用户带回产品的循环，例如看到好友动态、回应动态、再创造新的动态给别人看。 |
| 留存 | 用户在某个时间点之后继续返回产品的比例，例如 D1、D7、D30 留存。 |
| 流失 | 曾经活跃的用户停止返回产品，或逐渐变得不活跃。 |
| 护栏指标 | 在优化日活跃用户数（DAU）时保护用户体验的指标，例如通知关闭率或跳出率。 |

## 问题

如何提高 Facebook 的日活跃用户数（DAU）？

## 提示

想想哪些行为能促使用户每天回来，以及如何在不只依赖通知的情况下，强化或扩大这些用户循环。

## 为什么这题重要

这是一道典型的产品感和指标执行题。

面试官不是想听一串随机的增长技巧，而是想看你能否：

- 精确定义日活跃用户数（DAU）。
- 找出自然驱动日常使用的用户行为。
- 对日活跃用户数（DAU）问题做分群分析，而不是把所有用户混在一起看。
- 提出能提升长期产品价值的实验。
- 避免短期提升日活跃用户数（DAU）、长期伤害信任或满意度的做法。

对 Facebook 来说，日活跃用户数（DAU）是核心健康指标，因为日常使用反映了用户习惯、内容相关性、社交连接和用户感知价值。

## 回顾：指标题框架（GAME）

指标题框架（GAME）可以帮助你结构化思考用户增长和用户参与度。

| 步骤 | 在本题中的含义 |
| --- | --- |
| 目标（Goals） | 定义日活跃用户数（DAU）是什么，以及它由什么驱动。 |
| 行为（Actions） | 找出能提升日常使用频率的关键用户行为。 |
| 指标（Metrics） | 衡量这些行为，并拆分使用趋势。 |
| 评估（Evaluate） | 设计并测试能产生可持续影响的干预措施。 |

## 第一步：目标（Goals）

目标是提高每天回到 Facebook 的独立用户数量。

日活跃用户数（DAU）是核心健康指标，因为它反映：

- 习惯性使用：用户有理由经常回来。
- 内容相关性：用户看到的帖子、视频、小组或推荐内容值得他们花时间。
- 社交价值：用户通过评论、消息、反应、标签和好友动态维持关系。
- 用户满意度：用户觉得 Facebook 值得打开，而不是直接忽略。

关键细节是，我们不应该靠花招来提升日活跃用户数（DAU）。

不健康的日活跃用户数（DAU）增长可能来自：

- 垃圾式通知。
- 黑暗模式。
- 标题党内容。
- 低质量病毒循环。
- 提升打开次数但降低信任的功能。

健康的日活跃用户数（DAU）增长应该来自长期留存和产品价值的提升。

## 第二步：行为（Actions）

要提升日活跃用户数（DAU），先识别用户在觉得 Facebook 有价值时每天会做什么。

### 被动消费

被动消费指用户消费内容，但不一定创造内容。

例子包括：

- 滑动信息流。
- 观看 Stories。
- 观看 Reels 或视频。
- 阅读评论。
- 浏览小组。
- 查看 Marketplace 或活动。

这很重要，因为许多用户每天回来，主要是为了看看有什么新内容。

### 社交触发

社交触发是由其他人引发的互动。

例子包括：

- 回复评论。
- 回复消息。
- 查看标签。
- 接受好友请求。
- 查看生日提醒。
- 回复小组中的评论。

这些行为很强，因为 Facebook 是社交网络，不只是内容流。来自真实人的提醒通常比泛泛的内容推荐更有意义。

### 创作与贡献

创作和贡献会让生态更健康，因为它们为其他用户创造内容和信号。

例子包括：

- 发布 Stories。
- 分享内容。
- 对动态作出反应。
- 评论好友帖子。
- 创建小组帖子。
- 发布照片或生活动态。

这些行为很重要，因为当用户不仅消费内容，也创造让别人回来的理由时，DAU 才更容易持续。

### 通知驱动的重新参与

通知可以把用户带回来，但不应该成为唯一机制。

例子包括：

- 推送通知。
- 邮件提醒。
- 红点或角标。
- 站内提醒。
- 好友动态提醒。

风险在于通知疲劳。如果 Facebook 过度依赖通知，用户可能关闭通知、快速跳出，或者产生负面情绪。

### 有流失风险的用户

提升日活跃用户数（DAU）往往意味着识别那些曾经活跃、但正在离开的用户。

有用的流失风险信号包括：

- 每周会话数（sessions）减少。
- 信息流浏览深度下降。
- 有意义互动减少。
- 距离上次发帖、评论或发消息的时间变长。
- 通知响应率下降。
- 好友或小组活动减少。

对这类用户，产品应该恢复真实价值循环，而不是简单发送更多提醒。

## 第三步：指标（Metrics）

主指标是日活跃用户数（DAU），但它必须拆分，并用行为指标补充解释。

### 主指标

追踪：

- 日活跃用户数（DAU）：每日独立活跃用户数。

需要清楚定义“活跃”。例如，活跃用户可以是打开 Facebook 并执行了有意义行为的人，而不是仅仅发生后台同步或误触打开的人。

### 拆分日活跃用户数（DAU）

日活跃用户数（DAU）应按以下维度拆分：

- 用户群组（cohort）。
- 新用户与老用户。
- 被重新激活的用户。
- 平台：iOS、Android、移动网页、桌面网页。
- 地区。
- 账号年龄。
- 用户兴趣区域，例如小组、视频、好友、Marketplace、活动。
- 通知权限状态。

拆分很重要，因为日活跃用户数（DAU）问题很少是均匀的。新用户下降可能意味着新手引导（onboarding）有问题；老用户下降可能意味着信息流相关性或社交图谱疲劳。

### 核心行为指标

追踪每个用户核心行为的频率，例如：

- 发送消息数。
- 浏览 Stories 数。
- 浏览帖子数。
- 观看 Reels 数。
- 评论数。
- 反应数。
- 接受好友请求数。
- 浏览或创建小组帖子数。
- Marketplace 访问数。
- 活动回应数。

这些指标帮助解释日活跃用户数（DAU）为什么变化，而不仅仅是它是否变化。

### 通知指标

追踪：

- 推送通知点击率。
- 收到通知后的返回时间。
- 通知关闭率。
- 通知忽略率。
- 每条通知带来的会话数（sessions）。
- 由通知带回后的跳出率。

目标不是最大化通知数量，而是理解哪些通知能把用户带回有意义的体验。

### 留存指标

追踪：

- D1 留存：新用户一天后是否回来。
- D7 留存：用户一周后是否回来。
- D30 留存：用户一个月后是否回来。
- 曾经不活跃用户的重新激活率。

新用户引导（onboarding）要特别关注。如果新用户不能快速找到朋友、小组或自己关心的内容，未来日活跃用户数（DAU）会受限制。

### 高参与行为占比

追踪：

- 日活跃用户（DAU）中至少完成一次高参与行为的用户比例。

高参与行为可以包括：

- 发送消息。
- 发布 Story。
- 评论。
- 加入或发布小组内容。
- 分享内容。
- 回应活动。

这个指标可以区分浅层日活跃用户（DAU）和有意义的日活跃用户（DAU）。

### 流失与不活跃

追踪：

- 曾经活跃用户的流失率。
- 各用户群组（cohort）的不活跃率。
- 每周会话数（sessions）下降的用户比例。
- 7 天或 30 天内没有执行核心社交行为的用户比例。

这能帮助你找到日活跃用户数（DAU）在哪里流失。

## 第四步：评估（Evaluate）

要提高日活跃用户数（DAU），应优先关注投入相对低、影响可能高、并且能创造长期价值的实验。

### 实验 1：优化新手引导（onboarding）

假设：

如果新用户能更快理解产品的关键价值循环，他们会更频繁回来。

可能改动：

- 帮用户更快找到朋友。
- 在新手引导（onboarding）中推荐相关小组。
- 询问用户兴趣，并调整信息流推荐。
- 展示 Stories、小组和消息功能的使用方式。
- 鼓励用户在第一次会话（session）中完成第一个有意义行为。

指标：

- D1、D7、D30 留存。
- 首次好友连接时间。
- 首次有意义互动时间。
- 新用户对日活跃用户数（DAU）的贡献。
- 新手引导（onboarding）完成率。

护栏：

- 新手引导（onboarding）流失率。
- 用户反馈中的困惑程度。
- 推荐步骤跳过率。

### 实验 2：更智能的个性化通知

假设：

当通知与用户兴趣、关系和时区更匹配时，用户更可能回来。

可能改动：

- 发送更少但更相关的通知。
- 优先展示来自亲密好友或活跃小组的通知。
- 根据用户历史活跃时间发送通知。
- 避免短时间内堆叠太多通知。

指标：

- 通知点击率。
- 收到通知后的返回时间。
- 通知带回后的会话深度（session depth）。
- 返回后的高参与行为率。

护栏：

- 通知关闭率。
- 通知忽略率。
- 跳出率。
- 用户投诉或负面反馈。

### 实验 3：展示相关好友动态

假设：

当 Facebook 突出用户关心的人的活动时，用户会更频繁回来。

例子：

- “3 位好友发布了 Story。”
- “一位亲密好友评论了你的帖子。”
- “你的小组正在讨论你关注的话题。”
- “你的朋友正在参加附近活动。”

指标：

- 好友动态提醒后的返回率。
- Story 浏览量。
- 评论或反应数。
- 每用户会话数（sessions）。
- 日活跃用户（DAU）中发生社交互动的比例。

护栏：

- 隐藏或静音率。
- 用户感知到的打扰程度。
- 信息流满意度。

### 实验 4：突出未完成的互动循环

假设：

当 Facebook 让未完成的社交循环更可见、更容易完成时，用户更可能回来。

例子：

- 未读消息。
- 待处理好友请求。
- 评论回复。
- 等待审核的被标记照片。
- 有活跃回复的小组帖子。
- 未完成的草稿。

指标：

- 未完成循环完成率。
- 返回频率。
- 发送消息数。
- 评论数。
- 接受好友请求数。

护栏：

- 用户挫败感。
- 通知疲劳。
- 隐私担忧。

### 实验 5：提升内容相关性

假设：

如果信息流稳定展示相关内容，而不是低质量或重复内容，用户会每天回来。

可能改动：

- 更好地排序亲密好友内容。
- 推荐更相关的小组帖子。
- 减少标题党。
- 增加本地或兴趣相关推荐。
- 给用户更多信息流偏好控制。

指标：

- 信息流会话深度（session depth）。
- 高质量内容停留时间。
- 反应和评论。
- 隐藏和取关。
- D7 和 D30 留存。

护栏：

- 错误信息暴露。
- 低质量内容占比。
- 用户报告的信息流满意度。

## 如何排序实验优先级

我会按以下维度排序：

- 对日活跃用户数（DAU）的影响。
- 对假设的信心。
- 实现难度。
- 对用户信任的风险。
- 效果是否可能持续。

| 实验 | 可能影响 | 风险 | 原因 |
| --- | --- | --- | --- |
| 优化新手引导（onboarding） | 对新用户日活跃用户数（DAU）高 | 中 | 帮助未来日活跃用户数（DAU）基础，但需要谨慎设计流程。 |
| 智能通知 | 中到高 | 高 | 能带动返回，但过度使用会造成疲劳。 |
| 展示好友动态 | 高 | 中 | 利用 Facebook 的社交图谱，强化核心价值。 |
| 未完成互动循环 | 中 | 中 | 如果循环有意义会有效，过度强调会烦人。 |
| 内容相关性 | 高 | 高 | 与长期日活跃用户数（DAU）强相关，但执行更难。 |

## A/B 测试计划

对每个干预措施：

1. 定义目标用户分群（segment），例如新用户、有流失风险的用户、或活跃于小组的用户。
2. 随机分配用户到对照组和实验组。
3. 预先定义主指标和护栏指标。
4. 测试要运行足够长时间，捕捉留存影响，而不只是当天打开次数。
5. 按用户群组（cohort）、平台和地区分析异质性效果。
6. 只有在日活跃用户数（DAU）提升且没有伤害用户满意度或信任时，才上线。

## 强答案应该强调什么

强答案不应该只是说：“发更多通知。”

更强的答案应该说：

- 日活跃用户数（DAU）是每日价值循环的结果。
- Facebook 应该改善那些自然把人带回来的循环。
- 通知可以辅助，但只有当它指向有意义的内容或社交互动时才健康。
- 日活跃用户数（DAU）要做分群分析，这样才能判断问题来自新手引导（onboarding）、流失、内容相关性还是社交图谱活动。
- 实验必须包含通知疲劳、跳出率和内容质量的护栏指标。

## 30 秒回答

我会用指标题框架（GAME）提升 Facebook 日活跃用户数（DAU）。目标是通过真实产品价值增加每日回访用户，而不是靠花招。首先识别推动日常使用的关键行为：信息流和视频消费、评论和消息等社交触发、Stories 和帖子等创作行为，以及重新参与循环。指标上，我会按用户群组（cohort）、平台和地区拆分日活跃用户数（DAU），同时追踪第 1 天、第 7 天、第 30 天留存（D1/D7/D30 retention）、每用户核心行为频率、通知点击率、高参与行为占比，以及曾经活跃用户的流失情况。然后测试更好的新手引导（onboarding）、更智能的个性化通知、展示相关好友动态、突出未完成互动循环等干预，同时监控通知疲劳、跳出率上升和信息流满意度下降等护栏指标。
